# CustomButton 定制化按钮

## 场景

一般用于复杂业务组件实现，如果是基础按钮没有复杂样式，请用经过设计规范的 adapt.button 组件

### 以前的痛点

- 按钮按下、抬起、禁用等状态不好管理
- 按钮状态管理混乱，什么时候该播按下抬起动画没有统一的定义
- 事件定义（mouse/touch）不统一，在不同端上有不用表现
- 按钮click触发时机混乱，有些按钮长按也能触发，按下就能触发，按下拖动也能触发

## 实现方式/设计

分层设计，每个层级的默认宽高都和按钮宽高一致，并且是absolute元素，插槽可以直接相对定位。

组件中按钮分为四层：

- 文字之下
  - 阴影
  - 背景
  - 悬浮图标
- 文字层
- 文字之上
  - 动效（扫光啥的）
- 触发层: 点击区域

在状态变化时，组件根元素上会自动出现一些class，比如active，但是组件内并没有对这个class设置任何样式，这个类是留给使用用的，直接在选择器中搭配自动添加的class就好。

## 一些点

- pointer 事件实现，多端表现一致
- 自带防抖防暴击，默认200ms
- 自带扫光、呼吸（可选）
- 按下状态active已经管理好了，通过 `v-slot` 和 `defineExpose` 暴露给使用者使用
